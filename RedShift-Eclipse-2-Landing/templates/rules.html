{{ event_name }} Defense CTF

Формат проведения

Формат соревнования: каждая команда участников до 4 человек защищает свои сервисы, которые атакуют боты проверяющей платформы



Глоссарий

  • Инфраструктура: Организаторы предоставляют каждой команде одинаковую инфраструктуру, которую команды должны защищать
  • Сервисы: На VM каждой команды располагается ряд сервисов, каждый из которых содержит в себе заложенные уязвимости
  • Защита: Командам необходимо защищать свои сервисы от атак ботов, устраняя уязвимости
  • Атака: Атаки производят боты (скрипты) проверяющей платформы
  • Скоринг: Очки начисляются за успешную защиту своих сервисов, учитывается работоспособность сервисов и их компонентов

Условные обозначения

  • 😀 (Checker return a success state) - чекер выполнил проверку сервиса успешно
  • 😔 (Checker return a fail state) - чекер обнаружил проблему на сервисе, т.е. какой-то из компонентов сервиса работает некорректно
  • 💤 (Сhecker return an error state) - чекер не проверил функциональность по причине поломки проверяющего скрипта
  • 😈 (Sploit return a success state) - на сервисе успешно проэксплуатировала уязвимость
  • 😎 (Sploit return a fail state) - уязвимость не смогла быть воспроизведена на сервисе
  • 💥 (Sploit return an error state) - уязвимость не проверена про причине поломки проверяющего скрипта
  • 💀 (Service is down) - сервис мертв, т.е. сервис не прошел проверку ни одного из чекеров в рамках одного раунда

SLA

  • SLA (Service Level Agreement) - процент, получаемый по определенной формуле, обозначающий средний показатель жизни определенного сервиса команды - количество прошедших игровых раундов, которые сервис провёл со статусом checker success / количество прошедших раундов

    • sla = (a / (b * N)) * 100, где

    a - количество успешых sc для конкретного сервиса отдельной команды

    b - общее количество чекеров для отдельного сервиса

    N - количество прошедших раундов

    100 - для подсчета в процентах

    b * N - означает под собой SLA в 100%

    Технические ограничения

    • Запрещается DDoS, атаки на физическую инфраструктуру или сетевую инфраструктуру за пределами предоставленных машин и контейнеров
    • Командам разрешается изменять, добавлять или удалять любое программное обеспечение на своих машинах, если это не нарушает работу скоринговой системы или другие правила
    • Командам запрещается атаковать другие команды, даже в случае, если при атаках не происходит деструктива
    • Использование средств мониторинга трафика разрешено, если это не нарушает другие правила соревнований
    • Запрещается использование сторонних WAF и других подобных сторонних инструментов и систем, которые способны фильтровать трафик

    Этические и поведенческие правила

    • Уважение: Участники должны с уважением относиться к другим командам, судьям и организаторам соревнований
    • Честная игра: Не допускается предоставление ложной информации организаторам или попытки обмануть судей и систему оценки
    • Конфиденциальность: Команды не должны раскрывать найденные уязвимости до окончания соревнования.
